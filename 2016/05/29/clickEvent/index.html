tap原理解析

发表于 2016-05-29 | 分类于 事件机制

tap产生的原理

  • tap产生在开始触摸和结束触摸之间，按下抬起就是一个tap事件
  • 中间不能产生移动
  • tap时间差，200ms以内

tap 和 click 的区别

  • tap 只作⽤在移动端，PC端是⽆效的
  • click 在pc端和移动端都是ok的
  • 但是我们在移动端要⽤tap，因为 tap ⽐ click 快200-300ms

tap实战场景

  • 过节抢火车票
  • 抢红包

tap基本实现原理:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>tap</title>
    <meta name="viewport" content="width=device-width, user-scalable=no, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0">
    <style>
        .box{
            width: 200px;
            height: 200px;
            background-color: red;
        }
    </style>
</head>
<body>
<div class="box"></div>

<script>
    window.onload = function(){
        var box = document.getElementsByClassName('box')[0];
        box.onclick = function(){
          console.log('onclick');
        };
        var startTime = 0;  // 记录触摸开始时间
        var isMove = false; // 记录是否产生移动
        box.addEventListener('touchstart', function(){
            console.log('开始触摸');
            startTime = Date.now(); // 获取当前时间
        });
        box.addEventListener('touchmove', function(){
            console.log('开始移动====');
            isMove = true;
        });
        box.addEventListener('touchend', function(){
            console.log('结束触摸');
            if((Date.now() - startTime) < 200 && !isMove){
                // 触碰的时间在200ms以内，并且不产生移动
                console.log('产生了tap事件');
            }
            // 清零
            startTime = 0;
            isMove = false;
        });
    }
</script>
/body>
</html>
